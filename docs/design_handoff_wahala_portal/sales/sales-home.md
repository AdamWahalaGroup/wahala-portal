# Sales home / pipeline (frame 21)

> Part of the **Wahala Portal** design handoff — see [handoff index](../README.md). Visual reference: the labeled frames in `../Wahala Portal.dc.html`.

### 21 — Sales home / pipeline (`/dashboard/sales`, staff) — the Monday-meeting view
- Full AppShell (Sales→Board active). Header: kicker + "Pipeline" + Board/Leads/
  Proposals segmented tabs + **"+ Capture lead"** (ink).
- **4 stat cards**: Open pipeline $ (with weighted sub), Open deals, **Stuck 14d+**
  (amber, inset amber rail — prominent), Won/Lost this Q (green/red split + win rate).
- **New leads to qualify** strip (score chip + name + mono detail + Qualify/Pass),
  with a visible **"All leads →"** link (previously missed).
- **Pipeline = six stacked stage sections** (Discovery, Business req., Solution
  design, Proposal, Negotiation, Contract). Each header = color square + name +
  count pill + $ sum + a **probability anchor** ("anchor ≈40% → proposal"); the
  section with stuck deals turns amber and shows "N · M ⚠ stuck". Deal rows: name
  link · org/contact/owner mono · $ (tnum) · days/⚠-stuck tag · stage dropdown.
  Ends with a green **Won** strip (count + $ + view link).
