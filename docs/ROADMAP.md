# Wahala portal roadmap

## Now: first live deal

- Run the deposition-platform opportunity through the portal.
- Use the unified discovery review: paste evidence once, inspect every proposed
  change, and explicitly accept only supported Deal facts.
- Resolve IP ownership before proposal.
- Classify engagement, delivery, IP, data sensitivity, and support expectations.
- Record qualification evidence and one dated mutual commitment.
- Measure founder effort, AI spend, buyer delays, and delivery assumptions.

## Next: trustworthy commercial records

- Keep the user-facing `committed` stage labeled Contracting; migrate the
  internal enum only if the compatibility cost later becomes worthwhile.
- Add structured park and loss reasons.
- Make proposal response transitions atomic.
- Select and integrate a counsel-approved e-signature provider.
- Implement real payment checkout/webhooks and reconciliation.
- Add immutable agreement/proposal acceptance snapshots.
- Remove or production-gate hard-delete tools.
- Add rate limits and atomic one-use magic-link consumption.

## Then: management evidence

- Add date/cohort filters and sample sizes to scorecards.
- Track next-commitment coverage, stage conversion, cycle time, and forecast error.
- Add capacity estimates based on founder/engineering effort, not tokens.
- Add multi-tenant and Workers-runtime integration tests plus CI.

## After real outcomes: narrow AI expansion

- Build a golden evaluation set from accepted/rejected real examples.
- Use applied versus dismissed discovery suggestions to measure extraction
  quality by field; do not treat acceptance as proof of factual correctness.
- Evaluate discovery extraction, qualification evidence, meeting preparation,
  and proposal drafting independently.
- Add organization monthly spend caps and task-level model routing.
- Reconsider the MMA, capacity knobs, and pricing automation only after real
  usage identifies a stable decision they can improve.

## Parked

- Autonomous client communication
- Automatic opportunity loss or punt
- Token-based project estimates
- Artificial minimum delivery timelines
- Employee rankings based on AI scores or nudge obedience
- A second Deal/opportunity entity
